# Learning Loop (Mistakes + User Rejection)

## AI_MISTAKES.md 기록 규칙
기록은 짧고 재사용 가능해야 한다.
각 항목은 다음 형식을 따른다:
- 상황: (무슨 작업에서)
- 실수: (무엇을 잘못했나)
- 원인: (왜 그랬나)
- 다음엔: (예방 가드레일/체크리스트 1줄)
- 감지법: (테스트/검색/확인 커맨드 등)

## 유저 반려(Reject) 처리
유저가 “이 구현 싫음/틀림/다시” 라고 하면:
1) 방어하지 말고 반려 이유를 요구사항으로 재정의
2) 무엇이 어긋났는지 빠르게 3줄로 요약
3) AI_MISTAKES.md에 “반려 패턴” 항목 추가
4) TaskManager로 계획을 재분해하고, 가장 작은 atomic task부터 재시도
